Five major gaps closed in one release: file-based logging, a PR CI workflow, macOS builds, a security policy, and input validation. This is the release where the production scaffolding catches up to the quality of the core codebase. Two new issues also appear — the version filename mismatch and cubelit.key.pub returning to the repo.
New issues introduced in v0.1.3
⚠ Release asset filenames show v0.1.2 not v0.1.3
All installer assets are labelled 0.1.2 in their filenames. The version was not bumped in Cargo.toml and package.json before tagging, so Tauri named the builds after the old version. Users downloading the installer see the wrong version number.
⚠ cubelit.key.pub is back in the repo
The public key that was explicitly deleted in v0.1.2 has reappeared on master. It's a public key so not a secret, but it's confusing and suggests the root cause wasn't fixed. The permanent .gitignore fix comes in v0.1.4.
What was fixed in v0.1.3
✓
File-based logging added
App activity now logs to cubelit.log via the tracing crate. The biggest operational gap since launch is now closed. Crashes leave breadcrumbs for the first time.
✓
PR CI workflow added
cargo check, cargo clippy, cargo test, bun check, and bun test now run on every pull request. The test suite is now active, not passive.
✓
macOS Apple Silicon builds in CI
aarch64.dmg now ships with every release. macOS users have an official installer for the first time.
✓
SECURITY.md added
Supported versions, private vulnerability reporting channel, and a threat model. Proper security disclosure process now documented.
✓
Input validation on env vars
validate_env_vars helper checks length and NUL bytes before server creation and settings updates. Injection surface reduced.
✓
README game support clarified
Troubleshooting section added. Game support note added: "Only Minecraft Java and FiveM are fully supported in v0.1." The misleading table is finally fixed.
✓
3 pre-existing Clippy warnings fixed
Code quality cleaned up. authors field in Cargo.toml changed from "you" to "Cubelit Contributors".
