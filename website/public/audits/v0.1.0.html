Critical gaps at v0.1.0
Updater artifacts unsigned — auto-update is broken
The tauri-plugin-updater is wired but the signing key env var is wrong, so update checks will fail silently for any user who installs v0.1.0.
Zero tests
No unit tests, integration tests, or frontend tests of any kind. Every change ships blind.
No logging or observability
Crashes produce nothing. There is no way to diagnose a user-reported issue remotely.
README implies all 10 games work
The supported games table has no "Coming Soon" labels. A user trying to launch ARK or CS2 will silently fail with no explanation.
cubelit.key.pub committed to the repo
A signing key has no business being tracked in git. Creates confusion about which key is authoritative and risks future private key commits.
SQL strings at runtime, no migrations
Database schema lives in inline Rust strings. Bad queries only fail at runtime, not at compile time. No versioned migrations means schema changes are risky.
No PR CI workflow
The only automation is the release workflow. Regressions on branches go undetected until a tag is pushed.
No CHANGELOG or CONTRIBUTING.md
No record of what changed between versions. No guidance for contributors. Clone URL in README is a placeholder.
Where it stood at launch
v0.1.0 ships a genuinely working app — Minecraft and FiveM servers run, the Docker gate works, the UI is clean. But almost everything around the code — testing, CI, logging, signing, documentation — is absent or broken. The architecture is strong enough to build on. The question at this point was whether the scaffolding would be built systematically or left as-is.
