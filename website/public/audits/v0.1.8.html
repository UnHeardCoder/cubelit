↑ +3 from v0.1.7
v0.1.8 is a pure architecture release — and the most structurally significant in the project's history. The Rust backend was restructured from a monolithic src-tauri crate into a proper Cargo workspace with a new cubelit-core library crate. Business logic is now completely decoupled from Tauri. The EventSink trait decouples event emission. ServerRunner and ServerLifecycle traits create clean seams for future agents, CLI, and tests. The Minecraft Java recipe was also fixed to pin java25, preventing silent launch failures on all 1.21.10+ worlds.
What v0.1.8 actually means
Every release from v0.1.0–v0.1.7 was building toward this. The CI pipeline, logging, tracing spans, game registry — all of that was cleaning up the surface. v0.1.8 restructured the foundation. cubelit-core is now a pure Rust library with no Tauri dependency. src-tauri is a thin IPC adapter. This is the split that makes a CLI, an MCP server, or an HTTP agent all achievable without touching core logic. It's also the split that finally makes the command handler logic properly unit-testable.
Standout additions
cubelit-core Cargo workspace crate BIGGEST STRUCTURAL CHANGE YET
Business logic — error types, ports, recipes, DB queries, migrations, Docker orchestration, server lifecycle, RCON, backups — now lives in a pure Rust library crate with no Tauri dependency. src-tauri is now what it always should have been: a thin IPC adapter of ~5–15 lines per command. cubelit-core can be tested with cargo test independently of the Tauri runtime. This directly enables future CLI and HTTP/WebSocket agent transports without touching core logic.
EventSink trait — transport decoupling CLEAN ABSTRACTION
The core crate never calls app_handle.emit() directly anymore. Every long-running function takes &dyn EventSink. TauriEventSink in src-tauri maps CoreEvent variants to Tauri event names, byte-for-byte compatible with v0.1.7. NoopSink is available for tests. The wire format is locked down in a unit test (server_create_progress_wire_format_is_byte_identical_to_v0_1_7) — a regression fails CI before it ships.
ServerRunner + ServerLifecycle trait split SEAM FOR AGENTS
Narrow ServerRunner (pull image, create/start/stop/restart container, stats/logs) vs wide ServerLifecycle (create, delete, sync, rename, fetch, RCON, backup). LocalServerHost implements both for v0.1.8. Future remote agent support is now a matter of providing a ServerRunner impl that proxies to a remote Docker socket — the lifecycle layer doesn't change.
Minecraft Java pinned to java25 FIXES SILENT FAILURE
Vanilla 1.21.10+ ships a Java 25 bundler (class file version 69.0) and refused to start on the java21 image. Users would see a container that exits immediately with no meaningful error. This affects the most popular game mode and was completely invisible to the user — a high-value fix.
CI paths expanded for workspace layout CORRECTNESS
crates/**, Cargo.toml, Cargo.lock, .sqlx/ are now covered by the CI paths filter. Previously a change to cubelit-core alone might not have triggered CI — a serious gap given that core is now where all the logic lives.

Remaining gaps
No Sentry / crash reporting — longest-standing open gap
Users hit crashes with no telemetry path back to you. The EventSink architecture actually makes this easier now — a SentryEventSink impl could wrap the existing one. Still ~2 hours of work. Still the highest-priority observability gap.
No mock-based tests for ServerLifecycle / ServerRunner
The trait seam now exists. NoopSink exists. There is no architectural excuse remaining. Trait-based unit tests covering create_server, delete_server, and error paths with a mock ServerRunner are the #1 testing gap.
bollard 0.18 → 0.20.2 breaks CI (PR #27 red)
Bollard's streaming API has breaking changes in 0.20. The dep is stuck at 0.18 until the call sites in cubelit-core are manually updated. This needs fixing before it becomes a security or compatibility issue — don't let it sit.
8 open Dependabot PRs accumulating
Four opened in the last 20 hours alone. A weekly triage habit would prevent this from piling up. Most are safe to merge once bollard is unblocked.
Still 2 of 10 games
The registry refactor (v0.1.7) and the LocalServerHost lifecycle seam (v0.1.8) have removed every structural blocker. Valheim and Terraria are both simple Docker images. A recipe + dashboard card is now the only remaining work for each.
Dead code: stream_container_logs / ServerLogLine
Fine as forward scaffolding, but should be marked with #[allow(dead_code)] and a TODO comment before compiler warnings surface in CI as the project grows.
No Intel macOS build
Apple Silicon only. Low priority given market share trends but still a gap.
